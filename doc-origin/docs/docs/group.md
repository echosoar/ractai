# 群聊模式

## 介绍

把多个使用“不同模型”的“不同的Agent”，拉到一个群聊里面，发送的消息会有合适的智能体进行回复。

Agent 在回复群聊消息时，也会调用不同的 MCP、Skills。

![](/doc-static/usage/group-chat.png)

## 使用

在会话上点击右键，选择`开启群聊模式`：

![](/doc-static/usage/group-chat-open.png)

之后，在群聊消息的顶部，可以添加和配置群聊成员信息。

## 配置

在群聊中，决策哪些 Agent 来回复消息，有两种模式：`仲裁模式`（默认） 和 `自判断模式`，可以在 `设置` - `聊天设置` 中进行配置。

`仲裁模式` 指的是由一个单独的模型，来判断由哪个 Agent 来回复消息，因此可以在设置中设置仲裁的模型，默认为当前会话模型，也可以自定义配置。

`自判断模式` 指的是每个 Agent 都会对消息进行判断，认为自己可以回复的话，就会进行回复。
